Development environment this semester

We will use what could be called a combined development environment this semester.
This means that we will have three diffent ways of executing code.

  • Create script.py files and execute these files

  • Use the python interpretor (pyton shell)

  • Create .ipynb (jupyter notebook files)

.py files

. py files is normal text files where we write python code line by line.
This is the way you also did when writing java or javascript files. The file is executed by typing python script.py in the terminal, or by pressing the “play” botton.

The interpretor

The Python interpreter is a program that reads and executes Python code. Often referred to as the “Python shell,” it provides an interactive environment where you can write and run Python commands one at a time. This interactive mode is an excellent tool for learning, testing, and debugging. You will not create any lasting files in the interpretor. It is a “write one line of code and execute it” tool. This is alike the terminal in your browser which you probably have use when debugging javascript.

You can access the Python interpreter by typing python in your command-line interface (CLI).

python

Jupyter Notebook files

Jupyter notebook files are more like an IDE in it self. It can contain markdown cells and python code cells. The files have the extension .ipynb. We will use this in the last part of this semester. It is also what you will use in the elective Machine Learning.
